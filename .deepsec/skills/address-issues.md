# Addressing Issues

## Process

1. Ask the user to provide the issue they want to address. If the issue is already provided, skip this step.
2. Verify the issue is valid and actionable.
3. If the issue is valid and actionable, address the issue by making the necessary changes to the codebase and explain how you did it.
4. If the issue is not valid or actionable, explain why and suggest a solution.
