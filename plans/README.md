Direct runtime targets resolved: `react-router-dom@^7.17.0`, `@daytona/sdk@0.187.0` (list API migrated to async iterator in `convex/daytona.ts`), `mermaid@^11.15.0` (also forced on `streamdown`'s nested copy via an `overrides` entry, closing 4 moderate classDef/Gantt advisories), `shell-quote@^1.8.4` override (closed the lone critical). Remaining items are not reachable on Systify's runtime paths or are owned by upstream packages with no compatible patch:

1. **protobufjs (8) & @grpc/grpc-js (2)** — `@daytona/sdk › @opentelemetry/sdk-node › …`. Inert: `createDaytonaClient` never sets `otelEnabled`, so the OTel export path never runs. Patched versions are pinned inside Daytona's OTel stack; forcing them skews a stack the SDK ships and tests against.
2. **fast-xml-builder (1, high)** — `@daytona/sdk › @aws-sdk/lib-storage › … › fast-xml-parser`. Reachable only via Daytona's AWS S3 storage features, off Systify's provision/exec path. Fix needs a major `fast-xml-parser` bump under aws-sdk, risking S3 XML parsing.
3. **esbuild (1, high)** — `convex › esbuild`, `vite › esbuild`. Build tooling, not a runtime dep; owned by Convex and Vite.
4. **fast-uri (2, high)** — `eslint › … › ajv` and `shadcn › @modelcontextprotocol/sdk › ajv`. Dev/CLI tooling, not runtime; the shadcn path is plan 002's concern.
5. **ws (moderate)** — `convex › ws`, `@daytona/sdk › isomorphic-ws › ws`. Transitive, owned by Convex and Daytona; no direct dependency to bump.
6. **uuid (moderate)** — `mermaid › uuid`. Nested inside `mermaid@11.15.0`; the vuln only triggers for `v3/v5/v6` calls with a `buf` arg, which mermaid does not use. No mermaid release pins a patched uuid yet; a `uuid` major override is riskier than the moderate it would close.
